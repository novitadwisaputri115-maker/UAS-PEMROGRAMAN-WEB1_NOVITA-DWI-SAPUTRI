Pertemuan 1: Konsep Dasar Basis Data

Pada pertemuan ini, kita mempelajari dasar-dasar sistem basis data dan bagaimana ia berevolusi dari sistem pemrosesan file tradisional.

1. Data vs Informasi

Penting untuk membedakan kedua istilah ini dalam dunia basis data:

  • Data: Fakta mentah yang belum diolah (Contoh: Angka 200, Nama "Andi").
  • Informasi: Data yang sudah diproses dan memiliki arti bagi pengguna (Contoh: "Saldo rekening Andi adalah Rp 200.000").

2. Karakteristik Pendekatan Basis Data

Mengapa kita menggunakan basis data? Berikut adalah karakteristik utamanya:

• Sifat Menjelaskan Diri Sendiri (Self-describing)
Sistem mengandung database itu sendiri dan deskripsinya (metadata).
• Insulasi antara Program dan Data
Struktur data dapat diubah tanpa harus mengubah program aplikasinya.
• Berbagi Data (Data Sharing)
Data dapat diakses oleh banyak user secara bersamaan (multi-user).

3. Peran DBMS

Database Management System (DBMS) adalah perangkat lunak yang bertindak sebagai antarmuka antara user dan database. DBMS bertanggung jawab atas:

Keamanan Mengatur siapa saja yang boleh mengakses data.
Integritas Menjaga agar data tetap akurat dan konsisten.
Pemulihan Menyediakan mekanisme backup jika terjadi kegagalan sistem.

Pertemuan 3: Model Data Relasional

Model data relasional adalah standar industri saat ini di mana data disusun dalam bentuk tabel (relasi) dua dimensi yang terdiri dari baris dan kolom.

Tokoh Penting: Konsep ini diusulkan oleh Dr. E.F. Codd pada tahun 1970 dan menjadi dasar bagi DBMS modern seperti MySQL, PostgreSQL, dan Oracle.

1. Terminologi Tabel

Dalam dunia akademik, terdapat istilah formal untuk menyebut bagian-bagian tabel:

Istilah Umum Istilah Relasional Keterangan
Tabel Relasi Kumpulan data terorganisir
Baris Tupel Satu baris record data utuh
Kolom Atribut Karakteristik/properti data

2. Jenis-Jenis Key (Kunci)

Key digunakan untuk mengidentifikasi data secara unik dan menghubungkan antar tabel:

  • Super Key: Satu atau gabungan atribut yang dapat membedakan setiap baris secara unik.
  • Candidate Key: Super key dengan jumlah atribut paling minimal.
  • Primary Key: Kunci utama yang dipilih untuk mengidentifikasi baris (tidak boleh kosong/NULL).
  • Foreign Key: Atribut yang merujuk pada Primary Key di tabel lain (sebagai penghubung).

3. Aturan Integritas

Untuk menjaga validitas data, model relasional mengenal dua aturan utama:

  1. Integritas Entitas: Primary Key tidak boleh berisi nilai NULL (kosong).
  2. Integritas Referensial: Nilai Foreign Key harus ada di tabel asal (Primary Key) atau bernilai NULL.

4. Kardinalitas Relasi

Menjelaskan batasan jumlah hubungan antar entitas:

  • One-to-One (1:1): Contoh: Satu warga negara memiliki satu NIK.
  • One-to-Many (1:N): Contoh: Satu dosen mengajar banyak mahasiswa.
  • Many-to-Many (N:M): Contoh: Banyak mahasiswa mengambil banyak mata kuliah.

Pertemuan 4: Normalisasi Basis Data

Proses untuk meminimalisir redundansi data dan mencegah anomali.

1. 1NF (First Normal Form)

Menghilangkan grup berulang dan memastikan atomisitas data.

2. 2NF (Second Normal Form)

Memenuhi 1NF dan menghilangkan ketergantungan parsial.

Pertemuan 5: SQL - Data Definition Language

Digunakan untuk mendefinisikan struktur database.

CREATE TABLE Mahasiswa (
  nim CHAR(10) PRIMARY KEY,
  nama VARCHAR(100),
  jurusan VARCHAR(50)
);

Pertemuan 6: Data Manipulation Language (DML)

DML digunakan untuk memanipulasi data yang ada di dalam objek basis data (tabel). Setelah struktur tabel dibuat dengan DDL, kita menggunakan DML untuk mengelola isinya.

Penting: Perintah DML tidak mengubah struktur tabel, hanya mengubah isi (baris/record) di dalamnya.

1. Perintah INSERT

Digunakan untuk memasukkan baris data baru ke dalam tabel.

-- Menambah satu baris data
INSERT INTO Mahasiswa (nim, nama, jurusan)
VALUES ('202301001', 'Budi Santoso', 'Informatika');

2. Perintah SELECT

Digunakan untuk menampilkan data dari satu atau lebih tabel.

-- Menampilkan semua kolom dan semua baris
SELECT * FROM Mahasiswa;

-- Menampilkan kolom tertentu dengan kondisi
SELECT nama, jurusan FROM Mahasiswa
WHERE jurusan = 'Informatika';

3. Perintah UPDATE

Digunakan untuk mengubah data yang sudah tersimpan di tabel.

-- Mengubah jurusan mahasiswa tertentu
UPDATE Mahasiswa
SET jurusan = 'Sistem Informasi'
WHERE nim = '202301001';

*Hati-hati: Selalu gunakan klausa WHERE agar tidak mengubah seluruh baris data!

4. Perintah DELETE

Digunakan untuk menghapus satu atau lebih baris data.

-- Menghapus data mahasiswa berdasarkan NIM
DELETE FROM Mahasiswa
WHERE nim = '202301001';

Pertemuan 7: Keamanan Basis Data

Keamanan basis data mencakup perlindungan terhadap data dari akses yang tidak sah, kerusakan, atau modifikasi yang tidak diinginkan.

Prinsip Utama (CIA):
  • Confidentiality: Rahasia (Hanya user resmi yang bisa melihat).
  • Integrity: Data asli dan tidak dimanipulasi secara ilegal.
  • Availability: Data selalu siap saat dibutuhkan.

1. Tingkatan Keamanan

Keamanan tidak hanya di dalam database, tapi berlapis-lapis:

Level Penjelasan
Physical Keamanan lokasi server (Gedung, CCTV, Kunci pintu).
Network Penggunaan Firewall dan enkripsi saat transmisi data.
Operating System Keamanan akun login di komputer server.
Database System Pengelolaan User dan Password di dalam DBMS.

2. SQL Data Control Language (DCL)

Untuk mengelola akses user, kita menggunakan dua perintah utama:

A. GRANT (Memberikan Hak Akses)
-- Memberikan izin melihat tabel kepada user 'staff_admin'
GRANT SELECT, INSERT ON Mahasiswa TO staff_admin;
B. REVOKE (Mencabut Hak Akses)
-- Mencabut izin menghapus data dari user 'staff_admin'
REVOKE DELETE ON Mahasiswa FROM staff_admin;

3. Ancaman Basis Data: SQL Injection

Salah satu ancaman paling berbahaya adalah SQL Injection, di mana penyerang memasukkan kode SQL berbahaya melalui input form aplikasi.

Cara Mencegahnya:
  • Gunakan Prepared Statements atau Parameterized Queries.
  • Lakukan validasi input pada sisi aplikasi.
  • Gunakan prinsip Least Privilege (Beri hak akses seminimal mungkin).
